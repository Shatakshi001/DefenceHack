File Path System in HTML

HTML file paths specify the location of files or resources that a webpage needs to access, such as images, videos, scripts, or other HTML documents. These paths tell the web browser where to find the files required to display the content correctly or to execute scripts as intended.
To insert a file in a web page, its source must be known. For example, the syntax (<img src=" " alt=" ">) is used to insert an image file, where the path of the file is mentioned in the source (src).
The browser needs an exact, unambiguous address to locate the file. A file path is that address.

Types of File Paths

There are two main types of HTML File Paths:
  1. Relative File Paths
  2. Absolute File Paths

Relative File Paths(Most Important for the Projects)

Specify the path to a resource in relation to the location of the HTML file currently being viewed. Ideal for resources within the same website. Keeps your HTML portable if the domain changes since the path doesn't need to be updated.

Different Scenarios:

  • Linking to a file in the SAME Folder:

    You are in index.html and want to link to about.html.They are neighbors, living in the same my-website/ folder. The directions are as simple as possible.Just use the filename.

  • Linking to a file in a SUB-FOLDER (Going Down):

    You are in index.html and want to display logo.png. From index.html, you need to go into the images folder to find the logo. You write the folder name, a forward slash /, and then the filename.

  • Linking to a file in a PARENT FOLDER (Going Up):

    You are in contact.html (which is inside pages/) and you want to display logo.png (which is inside images/).  You can't go directly from pages/ to images/. You must first go up one level out of the pages folder to get back to the main my-website/ folder. From there, you can go down into the images folder. Two dots and a slash (../) means "go up one level. ../ takes you from pages/ up to my-website/. images/logo.png then takes you down into the images folder to find the file.

Absolute File Paths

An absolute file path gives the full, complete URL to a resource on the web. It starts with http:// or https://.
When to USE:
You ONLY use absolute paths when you are linking to a resource that is NOT on your own website.
  1. Linking to another website.
  2. Using a font from Google Fonts.
  3. Using an image from an image-hosting service.


HTML Boilerplate Code

When starting a new web project, setting up a solid foundation is crucial. An HTML boilerplate serves as this foundation, providing a standard starting point that ensures best practices and compatibility across different browsers.

What is HTML Boilerplate?

An HTML boilerplate is a set of pre-written HTML, CSS, and JavaScript files that form the starting point for your web project. It includes essential elements like the doctype declaration, meta tags, link to CSS files, and script tags for JavaScript. The primary goal is to save time and ensure consistency by providing a robust template that follows industry standards.

Why USE an HTML Boilerplate?

Using an HTML boilerplate has several benefits:
  • Efficiency: Quickly set up new projects without rewriting the same basic structure each time.
  • Consistency: Maintain a consistent structure and style across multiple projects.
  • Best Practices: Follow industry standards and best practices out of the box.
  • Cross-browser compatibility: Ensure your site works well across different browsers and devices.

The Anatomy of the HTML Boilerplate

Essential Elements of HTML Boilerplate:

  1. <!DOCTYPE html>
    This is the very first line and it's called the Document Type Declaration. In modern HTML5, it's incredibly simple. This specific line tells the browser: "Use the latest, standard-compliant mode for rendering this HTML document." It's a switch that prevents the browser from falling back into "quirks mode" (an old mode for rendering non-standard pages). It must always be the first thing in your file.
  2. <html lang="en">.....</html> This is the root element that wraps your entire page. The lang="en" attribute declares that the primary language of the page content is English. This is very important for accessibility and SEO. You'd change "en" to "es" for Spanish, "hi" for Hindi, etc.
  3. <head>.....</head> The <head> section is the container for this metadata. Nothing you put inside the <head> will be displayed in the main browser window.
  4. <meta charset="UTF-8"> This meta tag explicitly tells the browser to use UTF-8, which is the universal standard character set for the web. It can represent almost any character and symbol from any language in the world. This line is essential to prevent text-encoding issues.
  5. <meta name="viewport" ...">  This specific meta tag is the cornerstone of responsive design.
    • width=device-width: This tells the browser: "Make the page's width equal to the screen width of the device it's being viewed on."
    • initial-scale=1.0: This sets the initial zoom level to 100% when the page is first loaded.
    • In simple terms: this line tells the browser to render the page in a way that is optimized for mobile screens.
  6. <title>.....</title> The <title> tag sets the text that appears in the browser tab, in bookmark lists, and in search engine results. It's a critical piece of metadata for both usability and SEO.
  7. <body>.....</body> The <body> section is the container for everything the user will actually see on the page: your headings, paragraphs, images, links, tables, etc. All the tags you've learned so far go inside the <body>.

What is UTF-8?

UTF-8 is the character that represents each character using one to four bytes. UTF-8 is just like a dictionary which store the number corresponding to the character.
USE: Because ASCII can only store upto 256 characters of 1 byte. Widely used UTF-8. Support most of the character and display emojis.

Div Tag

The <div> tag, used in over 90% of web pages, divides content into sections and serves as a flexible container for text, images, and links. Easily styled with CSS and customized using classes or IDs, it often uses percentage-based widths like 50% or 33.33% for responsive layouts, making it key to web page structure.

class and id

The class attribute is used to specify one or more class names for an HTML element. The class attribute can be used on any HTML element. The class name can be used by CSS and JavaScript to perform certain tasks for elements with the specified class name. The class name in CSS stylesheet using "." symbol.
The id attribute is a unique identifier that is used to specify the document. It is used by CSS and JavaScript to perform a certain task for a unique element. In CSS, the id attribute is written using the # symbol followed by id.
